# Algoritmo-de-ordenamiento-Mezcla-Natural
Código de ordenamiento
ANÁLISIS DE COMPLEJIDAD:

Mejor caso
O(n)
Cuando los datos ya están casi ordenados, el algoritmo aprovecha las corridas naturales y reduce considerablemente el trabajo.

Caso promedio
O(n log n)
En una situación normal, el algoritmo necesita varias fusiones para completar el ordenamiento.

Peor caso
O(n log n)
Cuando los datos están completamente desordenados, el comportamiento es similar al Merge Sort tradicional.

CASOS DE USO:

Este método es eficiente cuando se trabaja con archivos grandes ya que se necesita una mayor eficiencia y rendiemiento que con los metodos simples y sobre todo cuando los numeros ya cuentan con un cierto orden previo.

COMPARATIVA CON OTRO MÉTODO (BURBUJA):

En el método de burbuja se compara elemento por elemento, requiere de muchas repeticiones y su complejidad es O(n²) siendo muy poco eficiente para datos grandes, en comparacion con mezcla natural que aprovecha las partes ya ordenadas, fusiona los bloques ya ordenados y es más eficiente con archivos o datos grandes.
